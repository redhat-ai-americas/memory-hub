# Benchmark Inventory

One section per benchmark reviewed. Category tags:

- **PM** = persistent-memory benchmark (tests an external memory layer across sessions)
- **LC** = long-context benchmark (tests how well the model uses a single large input; does **not** test persistent memory)
- **AG** = agent-capability benchmark (tests agentic behaviour where memory is one of several capabilities)
- **PS** = personalization / preference tracking
- **SURV** = survey / taxonomy, not a runnable benchmark

Long-context benchmarks are included here because they are routinely cited as "memory benchmarks" in practitioner discussions — the distinction needs to be explicit.

---

## Persistent-memory benchmarks

### LoCoMo — Evaluating Very Long-Term Conversational Memory of LLM Agents [PM]

- **Paper:** https://arxiv.org/abs/2402.17753 (Maharana et al., Snap Research)
- **Venue:** ACL 2024
- **Code/data:** https://github.com/snap-research/locomo ; https://snap-research.github.io/locomo/
- **What it measures:** QA over multi-session conversations between two fictional speakers, plus event summarization and multi-modal dialogue generation.
- **Corpus:** 1 986 QA pairs. 300 turns × up to 35 sessions per conversation, ~9K tokens avg. Generated by a machine-human pipeline with personas and temporal event graphs.
- **Question types:** single-hop, multi-hop, temporal, commonsense / world knowledge, adversarial.
- **Primary metric:** LLM-as-judge accuracy on QA.
- **Saturation status:** **Saturated** by frontier models with large context windows. Letta achieves 74.0% using just a filesystem and GPT-4o mini; Hindsight reports 92.0%. AMB authors explicitly say LoCoMo "can no longer tell systems apart" in the million-token era.
- **Open limitation:** chatbot-only; no agent tool use; no agent-environment interaction.

### LongMemEval — Benchmarking Chat Assistants on Long-Term Interactive Memory [PM]

- **Paper:** https://arxiv.org/abs/2410.10813 (Wu et al.)
- **Venue:** ICLR 2025
- **Code/data:** https://github.com/xiaowu0162/LongMemEval
- **What it measures:** **Five explicit long-term memory abilities:** information extraction, multi-session reasoning, temporal reasoning, knowledge updates, abstention.
- **Corpus:** 500 curated questions embedded in freely scalable user-assistant chat histories. Questions span six sub-types: single-session-user, single-session-assistant, single-session-preference, multi-session, knowledge-update, temporal-reasoning.
- **Primary metric:** accuracy per ability.
- **Known numbers:**
  - Commercial chat assistants and long-context LLMs drop ~30% accuracy vs short-context baselines (paper's headline).
  - Zep: up to +18.5% accuracy over baseline, **+38.4% on temporal reasoning** specifically, 90% latency reduction.
  - Hindsight on AMB leaderboard: 94.6%.
  - Hybrid-search baseline on AMB: 74.0%.
- **Why it matters:** **This is the most directly relevant academic benchmark for MemoryHub.** Four of its five abilities map onto capabilities we either ship or have on the roadmap (see `memoryhub-gap-analysis.md` for the mapping). "Abstention" is the one that we don't clearly test.
- **Saturation:** some saturation at the top, but the temporal-reasoning and knowledge-update sub-tasks still discriminate (per Zep's numbers).

### MemoryAgentBench (2025, ICLR 2026) [PM + AG]

- **Paper:** https://arxiv.org/abs/2507.05257 (Hu, Wang, McAuley — UCSD)
- **Code/data:** https://github.com/HUST-AI-HYZ/MemoryAgentBench ; https://huggingface.co/datasets/ai-hyz/MemoryAgentBench
- **What it measures:** Four core memory competencies:
  1. **Accurate retrieval**
  2. **Test-time learning** — agent can pick up new skills / facts mid-run
  3. **Long-range understanding**
  4. **Selective forgetting** — agent should not carry irrelevant or stale material
- **Tasks:** restructured existing datasets + two new ones:
  - **EventQA** — query over events accumulated across turns
  - **FactConsolidation** — reconcile facts across turns (update/supersede)
- **Why it matters for MemoryHub:** "selective forgetting" and "FactConsolidation" are directly in MemoryHub's wheelhouse — the curator-agent does inline dedup and there is an `update_memory` + version history + `report_contradiction` path. **This is the best candidate benchmark for MemoryHub's curator and versioning subsystems.**
- **Paper explicitly critiques** benchmarks that "rely on limited context lengths" or are "tailored for static, long-context settings like book-based QA".

### AMA-Bench — Long-Horizon Memory for Agentic Applications (Feb 2026) [PM + AG]

- **Paper:** https://arxiv.org/abs/2602.22769
- **What it measures:** Agent memory in **real agentic applications**, not chatbot dialogues. The authors' explicit framing: real memory is a stream of agent ↔ environment interactions dominated by machine-generated content, not human ↔ agent conversation.
- **Components:**
  1. Real-world agentic trajectories across representative applications + expert-curated QA
  2. Synthetic agentic trajectories scalable to arbitrary horizons + rule-based QA
- **Key finding:** existing memory systems underperform "because they lack causality and objective information and are constrained by the lossy nature of similarity-based retrieval." Their proposed AMA-Agent uses a causality graph + tool-augmented retrieval; achieves 57.22% average (vs strongest baseline +11.16%).
- **Relevance:** this is the benchmark that most directly punishes vector-similarity-only retrieval. MemoryHub is a vector-similarity-only retrieval system at its core (pgvector cosine + cross-encoder rerank + focus blend). **AMA-Bench would likely expose limitations in MemoryHub's retrieval model that LoCoMo/LongMemEval do not.**
- **Status on AMB leaderboard:** listed, 1 split × 0 runs as of April 2026.

### MemoryArena — Interdependent Multi-Session Agentic Tasks (Feb 2026) [PM + AG]

- **Paper:** https://arxiv.org/abs/2602.16313v1 — He, Wang et al. (MIT, UW, CMU, UCSD)
- **What it measures:** Agent memory across multiple *interdependent* sessions in four domains: web navigation, preference-constrained planning, information search, formal reasoning.
- **Design:** agents acquire memory while interacting with an environment, then reuse that memory in subsequent tasks whose solutions depend on earlier knowledge.
- **Key finding (quoted in user-provided source):** agents that perform well on long-context benchmarks perform poorly in MemoryArena's agentic setting. **Direct, independent corroboration that long-context ≠ persistent memory.**
- **Relevance:** matches the kind of agent-to-agent / cross-session workflow MemoryHub is designed to support (user setting focus, sub-agents writing memories that later sessions read). **If there is a public leaderboard or harness, running MemoryHub against this is high-leverage.** Needs follow-up to confirm code availability.

### MemGPT + DMR — Deep Memory Retrieval [PM]

- **Paper:** https://arxiv.org/abs/2310.08560 (Packer et al., UC Berkeley Sky Lab, Oct 2023)
- **MemGPT now part of Letta** (https://www.letta.com).
- **DMR:** a benchmark MemGPT introduced to measure cross-session consistency. Agent is asked questions whose answers require knowledge from earlier sessions.
- **Known numbers:** MemGPT 93.4%, Zep 94.8% (reported by Zep).
- **Saturation note:** Letta itself now argues DMR (and LoCoMo) is too easy, and has moved its own evaluation focus to **Context-Bench** (see below).

### Letta Context-Bench (Filesystem Suite + Skills Suite) [AG ≈ PM]

- **Home:** https://docs.letta.com/leaderboard (last updated 13 Mar 2026)
- **Repo:** https://github.com/letta-ai/letta-leaderboard ; newer harness https://github.com/letta-ai/letta-evals
- **Framing:** rather than evaluate isolated memory tools, evaluate models on how well they manage context end-to-end inside a fixed framework (Letta). This is a **model leaderboard**, not a memory-system leaderboard.
- **Filesystem Suite** — chain file operations, trace entity relationships, multi-step retrieval.
- **Skills Suite** — discover and load skills from a library.
- **Top scores (at 13 Mar 2026 snapshot):**
  - Filesystem Suite: `gpt-5.2-codex-xhigh` 93% @ $44.46 ; `gpt-5.4-xhigh` 89% @ $43.52 ; `claude-sonnet-4-6` 88% @ $116.09
  - Skills Suite: `gpt-5.2 xhigh` 85.31% / 63.12% ; `claude-opus-4-6` 81.32% / 62.64%
- **Cost column is first-class** on the leaderboard.
- **Caveat:** because Letta holds the framework constant, you cannot isolate "memory system X vs memory system Y". It tells you which model best uses a MemGPT-style hierarchy. That is different from what AMB measures.

### Agent Memory Benchmark (AMB) — Vectorize / Hindsight (Mar 2026) [PM harness]

- **Site:** https://agentmemorybenchmark.ai/
- **Code:** https://github.com/vectorize-io/agent-memory-benchmark
- **Launch post:** https://hindsight.vectorize.io/blog/2026/03/23/agent-memory-benchmark
- **What it is:** a harness that runs any memory provider through a four-step pipeline — **Ingest → Retrieve → Generate → Judge** — against multiple datasets. Generator and judge are both Gemini. Ingestion and retrieval time are measured separately; token cost is tracked.
- **Datasets bundled:** LoCoMo, LongMemEval, LifeBench, MemBench, MemSim, PersonaMem, BEAM, AMA-Bench.
- **Providers with scores at visit:** `hindsight`, `cognee`, `hybrid-search`, `bm25` (available as a baseline).
- **Modes:** `rag` (default top-k into prompt), `agentic-rag` (LLM can re-query), `agent` (bypass retrieval, call provider's own `direct_answer()`).
- **Why it matters for MemoryHub:** this is the **single best existing harness for running MemoryHub against a suite of memory benchmarks**. Building a MemoryHub provider adapter against AMB would produce a directly comparable score on LoCoMo, LongMemEval, LifeBench, PersonaMem, etc. in one go. The caveat is that AMB is a vendor leaderboard — currently every dataset's top score is the vendor who built the harness.

### AMB's new dataset: BEAM [PM]

- **Published only via AMB site as of this research pass**; I did not locate a separate arxiv paper.
- 100 conversations ranging from **100K to 10M tokens**; 2 000 questions across 10 memory ability categories.
- Hindsight scores across context sizes: 73.4% @ 100K, 71.1% @ 500K, 73.9% @ 1M, **64.1% @ 10M**.
- Hindsight claims the 10M sub-split is where accuracy gaps start to show — worth investigating whether the dataset/paper becomes public.

### LifeBench — Long-Horizon Multi-Source Memory [PM + PS]

- **Paper:** https://arxiv.org/abs/2603.03781 (Mar 2026)
- **What it measures:** long-term memory for personalized agents, targeting **non-declarative (habitual, procedural) memory** inferred from diverse digital traces — chats, calendar entries, notes, SMS, health records.
- **Explicit critique:** existing memory benchmarks mostly target declarative (semantic + episodic) memory.
- **Scope:** 10 users, multi-source.
- **Relevance:** interesting conceptually but niche — MemoryHub does not currently ingest non-declarative signals, and this benchmark's premise is that you pull memory from fragmented personal signals rather than an agent deliberately writing memories.

### MemBench — Comprehensive Memory Evaluation [PM]

- **Paper:** https://arxiv.org/abs/2506.21605 (Hu et al., ACL Findings 2025)
- **What it measures:** multi-aspect memory capability: effectiveness, efficiency, and capacity.
- **Memory types in dataset:** factual memory and reflective memory.
- **Scenarios:** participation and observation (different interactive modes).
- **Bundled in AMB.** Multiple choice format.

### MemSim — Bayesian Simulator for Memory Eval [PM]

- **Paper:** https://arxiv.org/abs/2409.20163 (Sept 2024)
- **What it is:** a Bayesian simulator (Bayesian Relation Network + causal generation) that automatically constructs QA pairs from generated user messages. Mitigates LLM hallucination in the factual base.
- **Generated dataset: MemDaily** — Chinese daily-life scenarios.
- **Relevance:** the methodology (simulator rather than static dataset) is interesting for our own internal evaluation — we could generate MemoryHub-shaped test data without writing QA pairs by hand.

### PersonaMem / PersonaMem-v2 [PS]

- **Paper (v1):** https://github.com/bowen-upenn/PersonaMem — COLM 2025.
- **Paper (v2):** https://arxiv.org/abs/2512.06688 (Dec 2025)
- **What it measures:** dynamic user profiling and personalized responses across long multi-session interactions.
- **v2 scope:** 1 000 personas, 20 000+ preferences, 300+ topics, up to 128K token context. Multimodal and multilingual.
- **Finding:** frontier LLMs achieve only **37–48% accuracy** on implicit personalization. "Reasoning remains the bottleneck for implicit personalization tasks."
- **Multiple choice** on AMB. Bundled there at "32k" and "128k" splits.
- **Relevance:** MemoryHub's scope system (user / project / role / organizational / enterprise) is designed for exactly this type of question (personal preference across sessions), but MemoryHub assumes explicit writes — it does not do implicit preference inference from unstructured chat.

## Long-context benchmarks (flagged: NOT persistent memory)

### Needle-in-a-Haystack (NIAH) [LC]

- https://github.com/gkamradt/LLMTest_NeedleInAHaystack (Kamradt, 2023)
- Place a sentence at varying depths in a long haystack, ask the model to recall it, sweep depth × length.
- **Tests retrieval inside a single prompt.** No persistent store, no cross-session anything.
- HELMET (below) showed vanilla NIAH is no longer predictive of downstream performance.

### RULER (2024) [LC]

- https://arxiv.org/abs/2404.06654 (Hsieh et al., NVIDIA)
- Extends NIAH with 13 tasks, including multi-hop tracing and aggregation, at variable sequence lengths.
- Finding: of 17 long-context models claiming ≥32K windows, only ~half maintain satisfactory performance at 32K in RULER.
- **Still a long-context benchmark.** Useful for choosing a backbone model, not for evaluating a memory layer.

### HELMET (2024) [LC]

- https://arxiv.org/abs/2410.02694 (Princeton NLP)
- https://github.com/princeton-nlp/HELMET
- Seven application-centric long-context categories at lengths up to 128K. Evaluated 59 LCLMs.
- Key finding: synthetic NIAH-style tasks don't predict downstream; open-source LCLMs lag significantly on full-context reasoning.
- **Long-context only.** Useful signal that benchmarks that look like they test memory (NIAH) don't even test what they claim to test, let alone persistent memory.

### BABILong (2024) [LC]

- https://arxiv.org/abs/2406.10149 (Kuratov et al. — NeurIPS 2024 D&B track)
- bAbI-style reasoning tasks with facts hidden in contexts up to 10M tokens.
- Finding: popular LLMs use only 10–20% of context effectively; RAG achieves ~60% on single-fact QA independent of context length.
- **Long-context only.**

### StreamBench (2024) [AG, online learning]

- https://arxiv.org/abs/2406.08747
- Evaluates continuous improvement under a stream of feedback (thumbs up/down). Tests online learning more than memory.
- **Tangential** — not a memory benchmark per se, but the feedback-stream design is relevant if MemoryHub later adds usage-signal weighting (which is on the roadmap per #61 focus history).

## Surveys and taxonomies (not runnable)

### "Memory in the Age of AI Agents: A Survey" (Dec 2025) [SURV]

- https://arxiv.org/abs/2512.13564 (Shichun Liu et al., 47 authors)
- Paper list: https://github.com/Shichun-Liu/Agent-Memory-Paper-List
- **Three-axis taxonomy:** Forms (token-level / parametric / latent) × Functions (factual / experiential / working) × Dynamics (formation / evolution / retrieval).
- Useful as a reference list and for positioning MemoryHub on the forms and functions axes. MemoryHub is unambiguously token-level form with factual + experiential functions.

### "Anatomy of Agentic Memory: Taxonomy and Empirical Analysis" (Feb 2026) [SURV]

- https://arxiv.org/abs/2602.19320 (Jiang et al., 22 Feb 2026)
- Four-structure taxonomy: Lightweight Semantic, Entity-Centric and Personalized, Episodic and Reflective, Structured and Hierarchical.
- **Critiques used heavily in this research:** "existing benchmarks are often underscaled, evaluation metrics are misaligned with semantic utility, performance varies significantly across backbone models, and system-level costs are frequently overlooked." Pain points: benchmark saturation, metric validity, judge sensitivity, backbone-dependent accuracy, latency/throughput overhead.
- MemoryHub sits in **Entity-Centric and Personalized** + **Structured and Hierarchical** on this taxonomy.
